# 도메인 기반 디렉토리 리팩토링 작업 이력

> 브랜치: `refactor/001-domain-base-refactor`

---

## 2026-03-05

### 도메인 기반 패키지 구조 리팩토링 실행

**요청**: 기존 레이어 기반(controller/, service/, repository/, domain/) 구조를 도메인 기반(student/, professor/, course/, enrollment/, common/) 구조로 전환하는 플랜 실행

**작업 내용**:

#### Phase 1: common/ 기반 구축
- `DomainBusinessException` 인터페이스 신규 생성
- ErrorCode, GlobalExceptionHandler, LockException, QueueException → `common/exception/` 이동
- GlobalExceptionHandler: 6개 개별 핸들러 → RuntimeException 통합 핸들러(DomainBusinessException 분기) 변경
- BaseEntity → `common/entity/`, CursorPageResponse/ErrorResponse → `common/dto/`
- Config, Lock, Queue, HealthCheck, Initializer 파일 이동
- JpaConfig에서 `@EnableJpaRepositories` 제거 (Spring Boot 자동 감지 위임)
- 커밋: `[refactor] 공통 모듈(common) 패키지 분리`

#### Phase 2: Student 도메인
- Student 관련 전체 파일 `student/` 패키지로 이동
- StudentService에 `existsById()`, `findStudentEntityById()` 메서드 추가
- EnrollmentService/TimetableService: StudentRepository → StudentService 의존성 변경
- 커밋: `[refactor] Student 도메인 패키지 분리`

#### Phase 3: Professor 도메인
- Professor 관련 전체 파일 `professor/` 패키지로 이동
- Course.kt에 Professor import 추가
- 커밋: `[refactor] Professor 도메인 패키지 분리`

#### Phase 4: Course 도메인
- Course, CourseSchedule, CourseType, VO, Repository, Service 등 `course/` 패키지로 이동
- ScheduleResponse를 CourseResponse에서 별도 파일로 분리 (SRP)
- EnrollmentService/TimetableService의 Course 관련 import 업데이트
- 커밋: `[refactor] Course 도메인 패키지 분리`

#### Phase 5: Enrollment 도메인
- Enrollment 관련 전체 파일 `enrollment/` 패키지로 이동
- EnrollmentStatus → 별도 파일 분리 (SRP)
- CancelEnrollmentRequest → 별도 파일 분리 (SRP)
- TimetableEntry → 별도 파일 분리 (SRP)
- 커밋: `[refactor] Enrollment 도메인 패키지 분리`

#### Phase 6: 정리
- 빈 디렉토리 삭제 (controller/, domain/, dto/, exception/, repository/, service/)
- TimetableResponse.kt 미사용 import 제거
- 커밋: `[refactor] 빈 디렉토리 정리 및 미사용 import 제거`

#### Phase 7: 테스트 재구성
- 11개 테스트 파일을 메인 소스와 동일한 도메인 구조로 이동
- IntegrationTestBase, MssTestApplicationTests, OpenApiGeneratorTest는 루트 유지
- 커밋: `[test] 테스트 디렉토리 도메인 기반 재구성`

**검증**: 각 Phase마다 `./gradlew compileKotlin` 통과 확인, Phase 7에서 `compileTestKotlin` 추가 확인

**비고**: ktlint Gradle 플러그인 미설치 상태로 ktlintFormat/ktlintCheck 스킵
